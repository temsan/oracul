# Oracul Bot - Telegram Self-Analysis Bot

Telegram бот для самоанализа диалогов с использованием AI и психологических фреймворков.

## 🎯 Возможности

- **💬 Самоанализ диалогов**: Анализ личных чатов и общения
- **🧠 Психологический анализ**: Big Five, архетипы Юнга, стили привязанности
- **🎤 Анализ голосовых**: Эмоциональное состояние через голосовые сообщения
- **📊 Каналы и группы**: Анализ активности в каналах и группах
- **💼 Карьерный анализ**: Поиск работы и карьерные рекомендации
- **🔮 Анализ ситуации**: Комплексный анализ текущего состояния

## 🚀 Быстрый старт

### 1. Настройка окружения
```bash
# Установка зависимостей
cd oracul-bot
pip install -r requirements.txt

# Настройка переменных окружения
cp .env.example .env
# Отредактируйте .env с вашими API ключами
```

### 2. Запуск бота
```bash
# Единая точка запуска (рекомендуется)
python run.py                 # Только бот
python run.py --webapp        # Бот + Web App
python run.py --webapp-only   # Только Web App

# Или напрямую:
cd oracul-bot && python unified_bot.py
```

### 3. Использование
1. Отправьте `/start` боту в Telegram
2. Выберите тип анализа из 6 категорий
3. Следуйте инструкциям бота

## 🔧 Конфигурация

### Переменные окружения (.env)
```bash
# Telegram API
TG_API_ID=your_telegram_api_id
TG_API_HASH=your_telegram_api_hash
BOT_TOKEN=your_bot_token

# OpenRouter API для LLM
OPENROUTER_API_KEY=your_openrouter_key

# Модели (приоритетный порядок)
DEFAULT_MODEL=openai/gpt-oss-120b:free
BACKUP_MODEL=openai/gpt-oss-20b:free
THIRD_MODEL=nvidia/nemotron-3-nano-30b-a3b:free
FOURTH_MODEL=arcee-ai/trinity-mini:free

# Настройки сессий
DEFAULT_SESSION_TTL_MINUTES=60
SESSION_TTL_OPTIONS_MINUTES=15,60,180,720,1440
DEFAULT_SESSION_MODE=persistent
SESSION_WEB_APP_URL=https://your-domain.com:8088  # опционально
```

### Поддерживаемые модели
- **OpenAI GPT-OSS**: 120B и 20B параметров (лучшее качество)
- **NVIDIA Nemotron**: 30B параметров (баланс скорость/качество)
- **Arcee Trinity**: Оптимизированная модель
- **Автоматический fallback** при rate limits

## 🔐 Система сессий

Каждый пользователь бота авторизуется через **свой** Telegram-аккаунт:

- **Отдельная сессия** — у каждого пользователя своя персональная сессия
- **Два режима работы:**
  - **TTL (постоянная)** — сессия с ограниченным временем жизни, автоматически продлевается
  - **Временный чат** — одноразовая сессия, удаляется после каждого запроса
- **Настраиваемое TTL** — от 15 минут до 1 дня
- **Web App** — веб-интерфейс для управления сессией (опционально)

### Как авторизоваться

1. Нажмите **🔐 Сессия** в главном меню или отправьте `/login`
2. Поделитесь номером телефона (кнопкой или текстом в формате `+79991234567`)
3. Введите код подтверждения из Telegram
4. Если появится запрос — подтвердите вход в официальном приложении Telegram (Настройки → Устройства)

**Важно:** Если Telegram заблокировал вход с нового устройства:
- Откройте Telegram на телефоне
- Перейдите в Настройки → Устройства
- Подтвердите вход

### Команды сессий
```
/login  — Войти в личную сессию
/logout — Выйти из сессии
```

## 🔒 Конфиденциальность

- **Локальная обработка**: Голосовые сообщения обрабатываются с CUDA
- **Защищенные соединения**: Все API вызовы через HTTPS
- **Изолированные сессии**: Данные каждого пользователя изолированы
- **Пользовательский контроль**: Полный контроль над анализируемыми данными
- **Авто-очистка**: Временные сессии автоматически удаляются

## 📈 Производительность

- **Автоматический fallback**: Переключение между LLM при ограничениях
