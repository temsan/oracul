## 🚀 Запуск

### Быстрая настройка:
```bash
# Создание .env файлов и директорий
python setup_env.py
```

### Backend:
```bash
cd backend/
# Настройте .env с:
# - DATABASE_URL (PostgreSQL)
# - OPENROUTER_API_KEY (для продвинутых анализов)
pip install -r requirements.txt
python -m app.main
```

### Telegram Bot:
```bash
cd telegram-bot/
# Настройте .env с:
# - BOT_TOKEN (от @BotFather)
# - BACKEND_API_URL=http://localhost:8000
# - TELEGRAM_API_ID и TELEGRAM_API_HASH (для автосбора)
pip install -r requirements.txt
python bot.py
```

### Получение API ключей:
- **OpenRouter API**: https://openrouter.ai/ (для продвинутых анализов)
- **Telegram Bot Token**: https://t.me/BotFather
- **Telegram API**: https://my.telegram.org/apps (для автосбора постов)

## 🔧 Дополнительные возможности

### Автоматический сбор данных из Telegram:
- Сбор сообщений пользователя из чатов
- Поиск постов в каналах
- Анализ публичных сообщений по username
- Интеграция с Telethon для работы с Telegram API

### Команды Telegram Bot:
- `/analyze` - запуск анализа с выбором способа сбора данных и полной авторизацией
- `/collect` - отдельная команда для тестирования автосбора
- `/logout` - выход из Telegram API авторизации
- `/status` - проверка прогресса анализа
- `/analyzers` - список всех доступных анализаторов
- `/help` - подробная справка

### Процесс авторизации Telegram API:
1. **Пользователь выбирает** "Автоматический сбор"
2. **Вводит номер телефона** в международном формате
3. **Получает SMS код** и вводит его
4. **При необходимости** вводит пароль 2FA
5. **Бот автоматически собирает** сообщения из чатов, каналов, публичных источников
6. **Запускается анализ** с собранными данными

### Хранение API ключей:
- **Backend**: `OPENROUTER_API_KEY` в `backend/.env`
- **Telegram Bot**: `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` в `telegram-bot/.env`
- Безопасное хранение через переменные окружения
