# Тестирование Oracul Bot

## Способы тестирования

### 1. Автоматическое тестирование (MCP-style)

**Файл:** `test_oracul_mcp.py`

Тестирует бота через реальный Telegram API (Telethon), имитируя действия пользователя.

```bash
# Первый запуск - авторизация
python -c "from telethon import TelegramClient; c = TelegramClient('test_oracul_session', 21834116, 'api_hash'); c.start()"

# Запуск тестов
python test_oracul_mcp.py
```

**Что тестирует:**
- ✅ /start команда
- ✅ /help команда  
- ✅ Кнопка "Сессия"
- ✅ Поток входа
- ✅ Меню категорий

**Требования:**
- TG_API_ID и TG_API_HASH в .env
- Активный Telegram аккаунт для тестирования
- Бот должен быть запущен

### 2. Функциональное тестирование (без Telegram)

**Файл:** `oracul-bot/test_bot_functions.py`

Проверяет структуру кода, импорты, логику без подключения к Telegram.

```bash
cd oracul-bot
python test_bot_functions.py
```

**Что тестирует:**
- ✅ Все импорты
- ✅ Инициализация бота
- ✅ Методы сессий
- ✅ Генерация меню
- ✅ Маршрутизация
- ✅ Хранилище сессий
- ✅ Web App
- ✅ Переменные окружения

### 3. Ручное тестирование через Telegram

1. Запусти бота:
   ```bash
   python run.py
   ```

2. В Telegram найди бота: `@oracul_analysis_bot`

3. Проверь команды:
   - `/start` — главное меню
   - `/help` — справка
   - `/login` — вход в сессию
   - `/logout` — выход из сессии

4. Проверь кнопки:
   - 🔐 Сессия → настройки сессии
   - 💬 Самоанализ диалогов → выбор чата
   - 🧠 Психологический анализ → типы анализа
   - 🎤 Анализ голосовых → меню голосовых
   - 📊 Каналы и группы → список каналов
   - 💼 Карьерный анализ → карьерные функции
   - 🔮 Анализ ситуации → комплексный анализ

5. Проверь сессии:
   - Нажми "Войти"
   - Введи номер телефона
   - Введи код из Telegram
   - Выбери режим (TTL/Временный)
   - Попробуй проанализировать чат

### 4. Тестирование Web App

```bash
cd oracul-bot
pip install fastapi uvicorn
uvicorn session_webapp:app --host 0.0.0.0 --port 8088
```

Открой: `http://localhost:8088?user_id=123`

Проверь:
- Загрузка сессии
- Изменение TTL
- Переключение режима
- Сохранение настроек

## Отчеты

После автоматического тестирования создается отчет:
- `test_report_YYYYMMDD_HHMMSS.md`

## CI/CD

Для автоматического тестирования в CI:

```yaml
# .github/workflows/test.yml
name: Test Oracul Bot

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Python
        uses: actions/setup-python@v2
        with:
          python-version: 3.10
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run function tests
        run: cd oracul-bot && python test_bot_functions.py
```

## Ожидаемое поведение

### ✅ Нормальная работа

| Действие | Ожидаемый результат |
|----------|---------------------|
| `/start` | Главное меню с 6 категориями |
| `🔐 Сессия` | Меню настроек сессии |
| `🔐 Войти` | Запрос номера телефона |
| Ввод номера | Отправка кода в Telegram |
| Ввод кода | Успешный вход, сообщение "Вход выполнен" |
| `📋 Мои чаты` | Список личных чатов |
| Выбор чата | Начало анализа |

### ⚠️ Известные особенности

- При первом входе с нового устройства Telegram может потребовать подтверждение в приложении
- Если сессия истекла — нужно войти заново
- В режиме "Временный" сессия удаляется после каждого анализа

## Отладка

При ошибках проверь:

1. **Логи бота** — в консоли где запущен бот
2. **Переменные окружения** — `.env` настроен?
3. **Telegram API** — TG_API_ID и TG_API_HASH верны?
4. **BOT_TOKEN** — токен бота действителен?

```bash
# Проверка .env
python oracul-bot/check_env.py
```
